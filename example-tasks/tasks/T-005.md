# T-005

    ```md
    Replace all `window.alert()` and `confirm()` with standard application dialog
    components using shadcn/ui.
    
    **Problem:**
    - Use of native browser `window.alert()` and `confirm()`
    - Interface inconsistent with application design system
    - Inferior user experience
    
    **Implemented solution:**
    1. **Created hooks:**
       - `web/src/hooks/useAlert.ts` - Manages simple alert state
       - `web/src/hooks/useConfirm.ts` - Manages confirmation state
    
    2. **Created components:**
       - `web/src/components/ui/AlertMessageDialog.tsx` - Alert dialog
       - `web/src/components/ui/ConfirmDialog.tsx` - Confirmation dialog
    
    3. **Updated files:**
       - `web/src/pages/extractions/Extractions.tsx` - 8 alerts and 1 confirmation
       - `web/src/pages/reconciliations/ReconciliationDetail.tsx` - 3 alerts
       - `web/src/pages/cupons/Cupons.tsx` - 2 alerts
       - `web/src/pages/cupons/CupomDetail.tsx` - 1 alert and 1 confirmation
       - `web/src/pages/admin/Companies.tsx` - 5 alerts
       - `web/src/pages/admin/Credentials.tsx` - 1 alert and 1 confirmation
       - `web/src/pages/admin/CompanyGroups.tsx` - 1 confirmation
    
    **Benefits:**
    - Interface consistent with design system (shadcn/ui)
    - Better user experience with accessible dialogs
    - Reusable and organized components
    - Responsive dialogs with smooth animations
    - Cleaner and more maintainable code
    ```
